# Security Policy

## Supported versions

Dependency Risk Radar is pre-1.0. Security fixes are provided on the latest `main` branch.

## Reporting a vulnerability

Please do not open public issues for suspected vulnerabilities.

Send a private report to the maintainers with:
- affected component and version
- reproduction steps or proof of concept
- impact assessment
- suggested remediation if available

If private security contact is not yet configured, open a private draft advisory in GitHub Security Advisories for this repository.

## Disclosure process

1. We acknowledge receipt within 3 business days.
2. We validate and triage severity.
3. We prepare a fix and regression tests.
4. We coordinate disclosure timing and publish release notes.
